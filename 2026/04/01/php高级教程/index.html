php date()函数

  • 作用

    • PHP date() 函数可把时间戳格式化为可读性更好的日期和时间

  • 参数介绍
    |参数|描述|
    |—|—|
    |format|必需.规定时间戳的格式|
    |timestamp|可选.规定时间戳.默认是当前的日期和时间|

php date() - 格式化日期

  • date() 函数的第一个必需参数 format 规定了如何格式化日期/时间。
  • 这里列出了一些可用的字符：
    • d - 代表月中的天 (01 - 31)
    • m - 代表月 (01 - 12)
    • Y - 代表年 (四位数)

php包含文件

  • 在php中,你看样子服务器执行php文件之前在该文件中插入一个文件的内容

  • include 和 require 语句用于在执行流中插入写在其他文件中的有用的代码。

  • include和require语句除了处理错误的方式不同之外,在其他方面都是相同的

    • require 生成一个致命错误（E_COMPILE_ERROR），在错误发生后脚本会停止执行。
    • include 生成一个警告（E_WARNING），在错误发生后脚本会继续执行。

  • 包含文件省去了大量的工作。这意味着您可以为所有网页创建标准页头、页脚或者菜单文件。然后，在页头需要更新时，您只需更新这个页头包含文件即可。

  • 语法:
    include 'filename';
    或者
    require 'filename';

php cookie

cookie是什么

  • cookie 常用于识别用户。cookie 是一种服务器留在用户计算机上的小文件。每当同一台计算机通过浏览器请求页面时，这台计算机将会发送 cookie。通过 PHP，您能够创建并取回 cookie 的值。

如何创建cookie

  • setcookie()函数用于设置cookie
  • 注释:setcookie() 函数必须位于 标签之前。
    • 语法:
      setcookie(name, value, expire, path, domain);
      • name(必需): Cookie 的名称。后续可通过 $_COOKIE[‘name’] 获取该值。名称只能包含字母、数字、下划线，不能包含空格或分号。
      • value(必需): Cookie 的值，存储在客户端。该值不能包含分号、逗号或空格（建议进行 URL 编码）。如果不希望发送值，可设为空字符串。
      • expire(可选): 过期时间，Unix 时间戳格式（如 time()+3600 表示一小时后过期）。如果省略或设为 0，则 Cookie 在浏览器会话结束时（关闭浏览器）失效。如果设为过去的时间，Cookie 会被立即删除。

session是什么

  • PHP session 变量用于存储关于用户会话（session）的信息，或者更改用户会话（session）的设置。Session 变量存储单一用户的信息，并且对于应用程序中的所有页面都是可用的。

销毁session

  • 如果您希望删除某些 session 数据，可以使用 unset() 或 session_destroy() 函数。
    unset() 函数用于释放指定的 session 变量：
  • 语法:
    <?php session_start(); if(isset($_SESSION['views'])) { unset($_SESSION['views']); } ?>
  • 您也可以通过调用 session_destroy() 函数彻底销毁 session：
    <?php session_destroy(); ?>
  • 注释: session_destroy() 将重置 session，您将失去所有已存储的 session 数据。
